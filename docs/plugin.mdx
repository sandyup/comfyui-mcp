`comfyui-mcp` isn't just an MCP server — it ships as a complete **Claude Code
plugin**. Installing the plugin gives Claude expert, model-specific knowledge
about ComfyUI so it picks the right sampler, CFG, resolution, and model files
without trial and error.

```bash
# In Claude Code
/plugin marketplace add artokun/comfyui-mcp
/plugin install comfy
```

### Pairs with the official Civitai MCP — bundled, headless

comfyui-mcp doesn't proxy Civitai — it composes with Civitai's
[official MCP server](https://mcp.civitai.com/mcp). **The plugin wires it for
you:** the bundled `.mcp.json` declares the Civitai remote server next to
comfyui, so `/plugin install comfy` gives you `mcp__civitai__*` automatically
— no `claude mcp add`, no API key for browsing (the `Authorization` header
defaults to an empty Bearer, which Civitai accepts for its read tools).

Civitai's `search_models` returns a model-version id; comfyui-mcp's
`download_civitai_model` takes exactly that id, fetches the file into your
local `models/`, and then `generate_image` or the sidebar panel wires it onto
the graph. The `civitai` skill teaches Claude the full discovery → download →
generate handoff (and falls back to HuggingFace search when the Civitai MCP
isn't connected).

**Optional API key.** To unlock your account context (favorites, posting,
gated/early-access downloads), set `CIVITAI_API_TOKEN` — the *same* variable
comfyui-mcp uses for its own downloads, so one secret powers both. Standalone
(non-plugin) MCP users can add the server manually:

```bash
claude mcp add --transport http civitai https://mcp.civitai.com/mcp \
  --header "Authorization: Bearer YOUR_CIVITAI_API_KEY"
```

## 11 slash commands

`/comfy:gen` (generate from a prompt), `/comfy:debug` (diagnose a failed
workflow), `/comfy:install` (install node packs), `/comfy:viz` (workflow →
Mermaid), `/comfy:batch`, `/comfy:compare`, `/comfy:convert`, `/comfy:gallery`,
`/comfy:recipe`, `/comfy:director`, `/comfy:node-skill`.

## 4 autonomous agents

- **comfy-explorer** — deep-dives a node pack's source to document it
- **comfy-researcher** — problem statement → ranked pack recommendations
- **comfy-debugger** — root-causes failed workflows from history + logs
- **comfy-optimizer** — VRAM and speed tuning for a given GPU

## Live graph editing — no API keys

<Note>
  The sidebar panel ships as its own pack — **live on ComfyUI-Manager / the [Comfy Registry](https://registry.comfy.org/nodes/comfyui-mcp-panel)** (search `comfyui-mcp-panel`). Full guide: [Sidebar Panel](/panel).
</Note>

With [channels mode](/configuration#channels--drive-the-comfyui-sidebar-panel-from-your-own-agent-session)
and the [comfyui-mcp-panel](https://github.com/artokun/comfyui-mcp-panel)
sidebar pack, your Claude session edits the live ComfyUI graph: add nodes,
wire slots, set widgets — all Ctrl+Z-undoable, all driven by your existing
Claude subscription rather than per-token API billing.

```bash
claude mcp add comfyui -- npx -y comfyui-mcp --channels
```

## Hooks

Four hooks keep generation sessions tight: **VRAM pre-flight checks** before
enqueueing a workflow, **job-completion notifications**, **auto-opening the
latest generated image**, and **save warnings** before destructive operations.
